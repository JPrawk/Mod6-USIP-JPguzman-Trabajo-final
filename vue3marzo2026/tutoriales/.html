BootstrapVueNext

¿Qué es BootstrapVueNext?

BootstrapVueNext es la próxima versión de BootstrapVue, una biblioteca de componentes de interfaz de usuario para Vue.js basada en Bootstrap 5. Ofrece una integración perfecta de los componentes de Bootstrap con Vue.js, lo que facilita la creación de interfaces de usuario modernas y receptivas.

Características principales

  • Componentes Vue.js: BootstrapVueNext proporciona una amplia gama de componentes de interfaz de usuario basados en Bootstrap 5, todos ellos optimizados para su uso con Vue.js.
  • Integración sin esfuerzo: Los componentes de BootstrapVueNext se integran fácilmente con proyectos Vue.js existentes, lo que te permite aprovechar la potencia de Bootstrap junto con la flexibilidad de Vue.js.
  • Personalización: BootstrapVueNext permite la personalización de componentes a través de propiedades y eventos, lo que te permite adaptar los componentes a tus necesidades específicas de diseño y funcionalidad.
  • Optimización de rendimiento: La biblioteca está diseñada para ofrecer un rendimiento óptimo, con componentes ligeros y una arquitectura optimizada para el rendimiento de aplicaciones Vue.js.

Empezando con BootstrapVueNext

Para empezar a usar BootstrapVueNext en tu proyecto Vue.js, simplemente añade el paquete a tu proyecto a través de npm o yarn:

npm i bootstrap bootstrap-vue-next --save
npm i unplugin-vue-components -D

Registrar en main.js/ts


// Importaciones bootstrap-vue-next
import {createBootstrap, BButton, BModal} from 'bootstrap-vue-next'
import 'bootstrap/dist/css/bootstrap.css'
import 'bootstrap-vue-next/dist/bootstrap-vue-next.css'

// Configuración global de bootstrap-vue-next
app.use(createBootstrap({components: true, directives: true}));
app.component('BButton', BButton);
app.component('BModal', BModal);

Axios

¿Qué es Axios?

Axios es una librería JavaScript basada en promesas que se utiliza para realizar solicitudes HTTP desde el navegador o desde Node.js. Es especialmente útil para realizar peticiones AJAX y comunicarse con servidores web para obtener o enviar datos.

Características principales

  • Simplicidad: Axios proporciona una API simple y fácil de usar para realizar solicitudes HTTP, lo que facilita el trabajo con APIs y servidores web.
  • Compatibilidad con promesas: Utiliza el sistema de promesas de JavaScript para manejar las solicitudes y las respuestas de manera asincrónica, lo que permite un código más limpio y conciso.
  • Interceptores de solicitudes y respuestas: Axios permite configurar interceptores para preprocesar las solicitudes antes de enviarlas y postprocesar las respuestas antes de que sean manejadas por tu aplicación.
  • Soporte para el navegador y Node.js: Funciona tanto en el navegador como en el entorno de Node.js, lo que lo hace versátil y adecuado para una amplia gama de aplicaciones web y proyectos.

Instalar en el proyecto VUE3

npm install --save axios vue-axios

Editar el archivo main.js/ts


// Importaciones axios
import axios from 'axios'
import VueAxios from 'vue-axios'

// Configuración global de axios
app.use(VueAxios, axios);

Tutorial de Variables de Entorno en Vue 3

Las variables de entorno son útiles para configurar valores que pueden cambiar entre diferentes entornos (desarrollo, producción, etc.). Vue 3 facilita la gestión de estas variables mediante archivos de configuración específicos para cada entorno.

Paso 1: Crear Archivos de Variables de Entorno

En la raíz de tu proyecto Vue, crea los siguientes archivos de configuración:

.env
.env.local
.env.development
.env.production

Estos archivos contendrán las variables de entorno específicas para cada entorno. Por ejemplo:

.env
VUE_APP_API_URL=https://api.example.com

.env.development
VUE_APP_API_URL=http://localhost:3000

.env.production
VUE_APP_API_URL=https://api.example.com

Paso 2: Acceder a las Variables de Entorno en Vue

En tu código Vue, puedes acceder a estas variables mediante process.env. Por ejemplo:

src/main.js
import { createApp } from 'vue'
import App from './App.vue'

console.log('API URL:', process.env.VUE_APP_API_URL)

createApp(App).mount('#app')

Paso 3: Usar Variables de Entorno en Componentes

También puedes usar las variables de entorno en tus componentes Vue:

src/components/ApiComponent.vue
<template>
  <div>
    <h2>API URL: {{ apiUrl }}</h2>
  </div>
</template>

<script>
export default {
  data() {
    return {
      apiUrl: process.env.VUE_APP_API_URL
    }
  }
}
</script>

Paso 4: Variables de Entorno en Scripts y CSS

Las variables de entorno también pueden ser utilizadas en scripts y estilos. Aquí tienes un ejemplo:

src/components/ScriptComponent.vue
<template>
  <div>
    <button @click="fetchData">Fetch Data</button>
  </div>
</template>

<script>
export default {
  methods: {
    fetchData() {
      fetch(process.env.VUE_APP_API_URL)
        .then(response => response.json())
        .then(data => console.log(data))
    }
  }
}
</script>

Paso 5: Reemplazar Variables de Entorno en la Construcción

Durante la construcción de tu aplicación, Vue reemplazará automáticamente las variables de entorno basándose en el entorno actual. Asegúrate de definir el entorno correcto al ejecutar los comandos de construcción:

npm run serve -- --mode development
npm run build -- --mode production

Conclusión

Las variables de entorno en Vue 3 son una herramienta poderosa para gestionar configuraciones específicas de cada entorno de una manera limpia y eficiente. Sigue estos pasos para integrar y usar variables de entorno en tu proyecto Vue 3.

Tutorial Completo de Vuex en Vue 3

Vuex es una biblioteca para gestionar el estado en aplicaciones Vue. Proporciona un almacén centralizado (store) para todos los componentes en una aplicación, con reglas que aseguran que el estado solo se puede mutar de una manera predecible.

Paso 1: Instalación de Vuex

Primero, instala Vuex en tu proyecto Vue 3. Puedes hacerlo mediante npm o yarn:

npm install vuex@next --save
// o
yarn add vuex@next

Paso 2: Configurar el Store de Vuex

Después de instalar Vuex, crea un archivo para definir tu store. Generalmente, este archivo se llama store.js o index.js dentro de una carpeta store:

src/store/index.js
import { createStore } from 'vuex'

const store = createStore({
  state: {
    count: 0,
    baseUrl: 'https://api.example.com'
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment')
    }
  },
  getters: {
    doubleCount(state) {
      return state.count * 2
    },
    getBaseUrl(state) {
      return state.baseUrl
    }
  }
})

export default store

Paso 3: Integrar Vuex en la Aplicación Vue

Luego, integra el store en tu aplicación Vue. Abre tu archivo main.js y agrega el store:

src/main.js
import { createApp } from 'vue'
import App from './App.vue'
import store from './store'

createApp(App)
  .use(store)
  .mount('#app')

Paso 4: Usar Vuex en Componentes

Ahora puedes acceder al store desde tus componentes. Aquí hay un ejemplo de cómo leer y modificar el estado del store, así como acceder a la URL base:

src/components/Counter.vue
<template>
  <div>
    <p>Count: {{ count }}</p>
    <p>Double Count: {{ doubleCount }}</p>
    <p>Base URL: {{ baseUrl }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState(['count']),
    ...mapGetters(['doubleCount', 'getBaseUrl']),
    baseUrl() {
      return this.getBaseUrl
    }
  },
  methods: {
    ...mapActions(['increment'])
  }
}
</script>

Paso 5: Estructurar el Store en Módulos

A medida que tu aplicación crece, es posible que quieras dividir tu store en módulos para mantener el código organizado. Aquí tienes un ejemplo de cómo estructurar módulos:

src/store/modules/counter.js
const state = {
  count: 0
}

const mutations = {
  increment(state) {
    state.count++
  }
}

const actions = {
  increment({ commit }) {
    commit('increment')
  }
}

const getters = {
  doubleCount(state) {
    return state.count * 2
  }
}

export default {
  state,
  mutations,
  actions,
  getters
}

src/store/modules/config.js
const state = {
  baseUrl: 'https://api.example.com'
}

const getters = {
  getBaseUrl(state) {
    return state.baseUrl
  }
}

export default {
  state,
  getters
}

src/store/index.js
import { createStore } from 'vuex'
import counter from './modules/counter'
import config from './modules/config'

const store = createStore({
  modules: {
    counter,
    config
  }
})

export default store

Paso 6: Acceder a los Módulos en Componentes

Después de dividir tu store en módulos, puedes acceder a los estados, getters, acciones y mutaciones de esos módulos en tus componentes:

src/components/Counter.vue
<template>
  <div>
    <p>Count: {{ count }}</p>
    <p>Double Count: {{ doubleCount }}</p>
    <p>Base URL: {{ baseUrl }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState('counter', ['count']),
    ...mapGetters('counter', ['doubleCount']),
    ...mapGetters('config', ['getBaseUrl']),
    baseUrl() {
      return this.getBaseUrl
    }
  },
  methods: {
    ...mapActions('counter', ['increment'])
  }
}
</script>

Conclusión

Vuex es una herramienta poderosa para gestionar el estado en aplicaciones Vue. Permite centralizar el estado y facilita su gestión mediante un conjunto claro de reglas y patrones. Siguiendo los pasos anteriores, puedes integrar y usar Vuex de manera efectiva en tus proyectos Vue 3, y compartir configuraciones como la URL base del backend entre tus componentes.

Vue 3 Toast Component Tutorial

Step 1: Create the Toast Component (Toast.vue)


      <template>
        <div v-if="visible" :class="['toast', type]" @click="hide">
          <slot></slot>
        </div>
      </template>

      <script>
      export default {
        props: {
          type: {
            type: String,
            default: 'info'
          },
          duration: {
            type: Number,
            default: 3000
          }
        },
        data() {
          return {
            visible: false,
            timeout: null
          };
        },
        mounted() {
          this.show();
        },
        methods: {
          show() {
            this.visible = true;
            if (this.duration > 0) {
              this.timeout = setTimeout(() => {
                this.hide();
              }, this.duration);
            }
          },
          hide() {
            this.visible = false;
            clearTimeout(this.timeout);
          }
        }
      };
      </script>

      <style scoped>
      .toast {

        cursor: pointer;

        transition: opacity 0.5s;
      }

Step 2: Create the Toasts Manager (ToastsManager.vue)


      <template>
        <div class="toasts-container">
          <Toast
            v-for="(toast, index) in toasts"
            :key="index"
            :type="toast.type"
            :duration="toast.duration"
            @click="removeToast(index)"
          >
            {{ toast.message }}
          </Toast>
        </div>
      </template>

      <script>
      import Toast from './Toast.vue';

      export default {
        components: { Toast },
        data() {
          return {
            toasts: []
          };
        },
        methods: {
          addToast(message, type = 'info', duration = 3000) {
            this.toasts.push({ message, type, duration });
          },
          removeToast(index) {
            this.toasts.splice(index, 1);
          }
        }
      };
      </script>

      <style scoped>
      .toasts-container {
        position: fixed;
        top: 10px;
        right: 10px;
        z-index: 9999;
        display: flex;
        flex-direction: column;
        align-items: flex-end;
      }
      </style>

Step 3: Create a Plugin for Easy Use


      // Create a file `toasts.js` in your project

      import { createApp } from 'vue';
      import ToastsManager from './components/ToastsManager.vue';

      let instance;

      export default {
        install(app) {
          const container = document.createElement('div');
          document.body.appendChild(container);
          const toastApp = createApp(ToastsManager);
          instance = toastApp.mount(container);

          app.config.globalProperties.$toast = {
            show(message, type = 'info', duration = 3000) {
              instance.addToast(message, type, duration);
            }
          };
        }
      };

Step 4: Use the Plugin in Your Main Application


      // In your main Vue file (e.g., `main.js`)

      import { createApp } from 'vue';
      import App from './App.vue';
      import toasts from './toasts';

      const app = createApp(App);

      app.use(toasts);

      app.mount('#app');

Step 5: Use the Toast Component in Your Application

Manejo de Rutas en Vue

Definir rutas


    const routes = [
      { path: '/', component: Home }, // ruta simple
      { path: '/user/:id', component: User }, // ruta dinamica
      { path: '/admin', component: Admin, meta: { requiresAuth: true, role: 'admin' } }, //ruta con guardian
    ];

Definir Guard 


    router.beforeEach((to, from, next) => {
      const userRole = 'user'; // Simulación del rol del usuario (podría venir de un store o auth service)
      if (to.matched.some(record => record.meta.requiresAuth)) {
        if (to.meta.role === userRole) {
          next();
        } else {
          next('/'); // Redirigir a la página de inicio si no tiene permisos
        }
      } else {
        next(); // Asegúrate de llamar a next()
      }
    });
