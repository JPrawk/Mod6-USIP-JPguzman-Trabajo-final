Vue.js es un poderoso framework de JavaScript que se centra en la construcción de interfaces de usuario interactivas y dinámicas para aplicaciones web de una sola página (SPA).

Características principales de Vue.js

  • Reactividad: Vue.js utiliza un sistema de reactividad para mantener sincronizados los datos del modelo y la vista.
  • Componentes: Puedes crear componentes reutilizables que encapsulan la funcionalidad y la apariencia de una parte de la interfaz de usuario.
  • Directivas: Vue.js proporciona un conjunto de directivas integradas que te permiten agregar funcionalidad interactiva de forma declarativa.
  • Enrutamiento: Vue.js proporciona una solución de enrutamiento integrada (vue-router) para crear aplicaciones de una sola página con múltiples vistas.
  • Gestión del estado: Vue.js no impone una solución específica para la gestión del estado, lo que te permite elegir entre diferentes herramientas y patrones según tus necesidades.

En resumen, Vue.js es un framework flexible y fácil de usar que te permite crear aplicaciones web modernas y potentes con facilidad. Ya sea que estés construyendo una pequeña aplicación de una sola página o una aplicación web empresarial compleja, Vue.js ofrece las herramientas y la escalabilidad necesarias para llevar tus ideas al siguiente nivel.

Guía de Instalación del CLI de Vue 3

Paso 1: Instalación de Node.js y npm

Antes de instalar el CLI de Vue 3, necesitas tener Node.js y npm instalados en tu sistema. Puedes descargar e instalar Node.js desde su sitio web oficial.

Paso 2: Instalación del CLI de Vue

Una vez que tengas Node.js y npm instalados, puedes instalar el CLI de Vue 3 globalmente utilizando npm. Abre tu terminal y ejecuta el siguiente comando:

npm install -g @vue/cli

Este comando instalará el CLI de Vue 3 de forma global en tu sistema.

Paso 3: Verificar la Instalación

Una vez completada la instalación, puedes verificar que el CLI de Vue se haya instalado correctamente ejecutando el siguiente comando en tu terminal:

vue --version

Este comando debería mostrar la versión del CLI de Vue instalada en tu sistema.

Paso 4: Crear un Nuevo Proyecto Vue

vue create nombre-del-proyecto

Este comando te guiará a través de un proceso interactivo para configurar y crear un nuevo proyecto Vue.







Paso 5: Levantar el servidor de prueba

¡Felicidades! Ahora estás listo para crear un nuevo proyecto Vue utilizando el CLI de Vue 3. Para correr el proyecto, simplemente ejecuta el siguiente comando en tu terminal:

npm run serve

Estructura de Archivos de un Proyecto Vue 3

.
    ├── node_modules
    ├── public
    │   ├── favicon.ico
    │   └── index.html
    ├── src
    │   ├── assets
    │   ├── components
    │   │   └── HelloWorld.vue
    │   ├── App.vue
    │   └── main.js
    ├── .gitignore
    ├── babel.config.js
    ├── package.json
    └── README.md

src/

  • assets/: Directorio para archivos estáticos como imágenes, fuentes, etc.
  • components/: Directorio para componentes Vue reutilizables.
  • views/: Directorio para vistas o páginas de la aplicación.
  • App.vue: Componente raíz de la aplicación que contiene la estructura base de la aplicación.
  • main.js: Punto de entrada de la aplicación donde se crea la instancia de Vue y se monta en el DOM.
  • router.js: Configuración del enrutador (si se está utilizando vue-router).

public/

  • index.html: Archivo HTML principal de la aplicación donde se monta la aplicación Vue.
  • favicon.ico: Icono de la aplicación.
  • ...: Otros archivos estáticos como imágenes, archivos CSS, etc., que se sirven directamente.

node_modules/

  • ...: Dependencias del proyecto instaladas a través de npm o yarn.

package.json

  • Archivo de configuración de npm que incluye metadatos del proyecto y las dependencias del proyecto.

package-lock.json

  • Archivo generado automáticamente que almacena la versión exacta de cada dependencia instalada.

babel.config.js

  • Configuración de Babel para transpilar el código JavaScript de la aplicación.

webpack.config.js

  • Configuración de Webpack para empaquetar y procesar los archivos de la aplicación.

README.md

  • Archivo de documentación que describe el proyecto, cómo instalarlo y cómo usarlo.

Archivo main.js

¿Qué es el archivo main.js en Vue 3?

El archivo main.js es el punto de entrada principal de una aplicación Vue 3. Es el archivo donde se configura y se inicializa la instancia de la aplicación Vue, y se definen configuraciones globales como la integración con plugins, bibliotecas externas y componentes globales.

Partes del Archivo main.js


// Importaciones //

import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'

// Creación de la aplicación Vue //

const app = createApp(App);

// Configuración globales //

app.use(store);
app.use(router);

// Montaje de la aplicación //

app.mount('#app');

Descripción de las Partes

1. Importaciones

En esta sección, importamos los módulos necesarios para nuestra aplicación.


import { createApp } from 'vue'
import App from './App.vue'
import './registerServiceWorker'
import router from './router'
import store from './store'

2. Creación de la aplicación Vue

En esta sección, creamos una instancia de la aplicación Vue utilizando el componente principal.

const app = createApp(App);

La constante app ahora contiene la instancia de la aplicación Vue creada con el componente principal App.

3. Configuración globales


app.use(store);
app.use(router);

Utilizamos el método use para integrar los plugins en nuestra aplicación Vue.

3. Montaje de la aplicación

En esta última sección, montamos la aplicación en el DOM.

app.mount('#app');

Montamos la aplicación en el elemento HTML con el id app. Esto indica a Vue que tome el control de ese elemento y renderice la aplicación dentro de él.

Tutorial Completo de un Archivo .vue en Vue 3

Un archivo .vue es un componente monolítico que puede contener el marcado, la lógica y los estilos de un componente Vue. Vamos a desglosar un archivo .vue y todas las opciones que se pueden usar en la sección export default.

Estructura de un Archivo .vue

Un archivo .vue típico tiene tres secciones principales:

  • <template>: Contiene el HTML del componente.
  • <script>: Contiene la lógica de JavaScript del componente.
  • <style>: Contiene los estilos CSS del componente.
<template>
  <div>
    <h1>{{ message }}</h1>
  </div>
</template>

<script>
export default {
  name: 'MiComponente',
  data() {
    return {
      message: 'Hola Mundo'
    }
  },
  components: {
    // Registro de componentes que se utilizaran.
  },
  methods: {
    // métodos que se pueden llamar desde la plantilla o desde otras partes del componente.
  },
  computed: {
    // propiedades computadas que dependen de otras propiedades reactivas
  },
  props: {
    // propiedades que el componente puede recibir.
  },
  emits: [] // los eventos personalizados que el componente puede emitir.
}
</script>

<style>
h1 {
  color: #42b983;
}
</style>

Opciones en el export default

La sección export default define la lógica y el comportamiento del componente. A continuación, describiremos las principales opciones que puedes utilizar:

1. name

Define el nombre del componente. Es útil para el depurador y para el uso de componentes en la plantilla.

export default {
  name: 'MiComponente'
}

2. components

Permite registrar componentes hijos que se utilizarán dentro de este componente.

import OtroComponente from './OtroComponente.vue';

export default {
  components: {
    OtroComponente
  }
}

3. props

Define las propiedades que el componente puede recibir. Puede ser un arreglo de nombres de propiedades o un objeto con validaciones.

export default {
  props: {
    titulo: {
      type: String,
      required: true
    },
    contador: {
      type: Number,
      default: 0
    }
  }
}

4. data

Una función que retorna un objeto con el estado reactivo del componente.

export default {
  data() {
    return {
      mensaje: 'Hola Mundo',
      contador: 0
    }
  }
}

5. computed

Define propiedades computadas que dependen de otras propiedades reactivas.

export default {
  data() {
    return {
      contador: 0
    }
  },
  computed: {
    dobleContador() {
      return this.contador * 2;
    }
  }
}

6. methods

Define métodos que se pueden llamar desde la plantilla o desde otras partes del componente.

export default {
  methods: {
    incrementar() {
      this.contador++;
    }
  }
}

7. watch

Observadores que ejecutan funciones en respuesta a cambios en propiedades específicas.

export default {
  data() {
    return {
      contador: 0
    }
  },
  watch: {
    contador(nuevoValor, viejoValor) {
      console.log('Contador cambiado de', viejoValor, 'a', nuevoValor);
    }
  }
}

8. lifecycle hooks

Define funciones que se ejecutan en diferentes etapas del ciclo de vida del componente (creación, montaje, actualización, destrucción).

export default {
  created() {
    console.log('Componente creado');
  },
  mounted() {
    console.log('Componente montado');
  },
  updated() {
    console.log('Componente actualizado');
  },
  destroyed() {
    console.log('Componente destruido');
  }
}

12. emits

Define los eventos personalizados que el componente puede emitir.

export default {
  emits: ['miEvento']
}

Conclusión

Este tutorial cubre las principales opciones que puedes utilizar en la sección export default de un archivo .vue. Vue 3 es muy flexible y potente, lo que te permite construir componentes dinámicos y reutilizables de manera eficiente.

Interpolación en Vue.js

La interpolación en Vue.js es una técnica para vincular los datos de tu aplicación a la interfaz de usuario. Vue.js proporciona varias maneras de hacer esto usando la sintaxis de interpolación.

Interpolación simple



<template>
  <div>
    <h2>Simple Interpolation</h2>
    <p>{{ message }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, Vue 3!'
    };
  }
}
</script>

Interpolación de objetos



<template>
  <div>
    <h2>Object Interpolation</h2>
    <p>Name: {{ user.name }}</p>
    <p>Age: {{ user.age }}</p>
    <p>Email: {{ user.email }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      user: {
        name: 'John Doe',
        age: 30,
        email: 'john.doe@example.com'
      }
    };
  }
}
</script>

Interpolación de expresión



<template>
  <div>
    <h2>Expression Interpolation</h2>
    <p>{{ 1 + 1 }}</p>
    <p>{{ message.split('').reverse().join('') }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, Vue 3!'
    };
  }
}
</script>

Interpolación de atributos



<template>
  <div>
    <h2>Attribute Interpolation</h2>
    <img :src="imageSrc" alt="Vue logo">
  </div>
</template>

<script>
export default {
  data() {
    return {
      imageSrc: 'https://vuejs.org/images/logo.png'
    };
  }
}
</script>

Interpolación de texto



<template>
  <div>
    <h2>Text Interpolation</h2>
    <p v-text="message"></p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, Vue 3!'
    };
  }
}
</script>

Directivas en Vue.js

Las directivas en Vue.js son atributos especiales con el prefijo v-. Estas directivas proporcionan funcionalidad reactiva a tus aplicaciones, permitiendo enlazar datos, manejar eventos, y más. A continuación, se describen las principales directivas de Vue.js con ejemplos.

v-html

Renderiza HTML crudo dentro de un elemento. Ten en cuenta que puede ser un riesgo de seguridad si el HTML no es seguro.

<div v-html="htmlContent"></div>

Ejemplo

Crea un archivo src/views/HtmlRenderingView.vue con el siguiente contenido:


<template>
  <div>
    <h1>Rendering HTML with v-html</h1>
    <div v-html="rawHtml"></div>
    <button @click="updateHtml">Update HTML Content</button>
  </div>
</template>

<script>
export default {
    data() {
        return {
            rawHtml: '<p>This is some <strong>initial</strong> HTML content.</p>',
        };
    },
    methods: {
        updateHtml() {
            this.rawHtml = '<p>The HTML content has been <em>updated</em>!</p>';
        },
    },
};
</script>

<style scoped>
h1 {
    color: #42b983;
}
button {
    margin-top: 10px;
}
</style>

Agregar la ruta en src/router/index.js


import HtmlRenderingView from '../views/HtmlRenderingView.vue';

const routes = [
  {
    path: '/html-rendering',
    name: 'HtmlRendering',
    component: HtmlRenderingView,
  }
];

Agregar en el menu src/App.vue


| <router-link to="/html-rendering"> HTML Rendering </router-link>

v-bind

Vincula un atributo HTML o una propiedad de un componente a una expresión en Vue. La sintaxis abreviada es :.

<img v-bind:src="imageSrc">
<img :src="imageSrc"> <!-- Sintaxis abreviada -->

Ejemplo:

Crea una nueva vista en src/views/UserCardView.vue para mostrar el componente UserCard. 

<template>
  <div>
    <div class="user-card" :style="{ borderColor: user.color }">
      <h2>{{ user.name }}</h2>
      <p>Email: <a :href="`mailto:${user.email}`">{{ user.email }}</a></p>
      <p>Age: {{ user.age }}</p>
      <img :src="user.avatar" :alt="`${user.name}'s avatar`" />
    </div>
  </div>
</template>

<script>
export default {
    name: 'UserCardView',
    data() {
        return {
            user: {
                name: 'John Doe',
                email: 'john.doe@example.com',
                age: 28,
                avatar: 'https://via.placeholder.com/150',
                color: 'blue'
            }
        };
    },
    mounted() {
        this.addStyles();
    },
    methods: {
        addStyles() {
            const style = document.createElement('style');
            style.type = 'text/css';
            style.innerHTML = `
        .user-card {
          border: 2px solid;
          padding: 16px;
          border-radius: 8px;
          max-width: 300px;
          text-align: center;
        }
      `;
            document.head.appendChild(style);
        }
    }
}
</script>
En este componente, utilizamos v-bind (abreviado como :) para enlazar dinámicamente los atributos style, href, src, y alt a las propiedades del objeto user.

  {
    path: '/user',
    name: 'UserCard',
    component: UserCardView
  }
];

v-if

Renderiza elementos de manera condicional basado en la verdad de la expresión.

<p v-if="seen">Ahora me ves</p>

v-else

Debe seguir a un v-if o v-else-if para renderizar un bloque "else".

<p v-if="seen">Ahora me ves</p>
<p v-else>Ahora no me ves</p>

v-else-if

Condición "else if" para v-if.

<p v-if="type === 'A'">A</p>
<p v-else-if="type === 'B'">B</p>
<p v-else>C</p>

Ejemplo

Crea un archivo src/views/ConditionalView.vue con el siguiente contenido:


<template>
  <div>
    <h1>Conditional Rendering with v-else-if</h1>
    <button @click="setView('first')">Show First</button>
    <button @click="setView('second')">Show Second</button>
    <button @click="setView('none')">Show None</button>
    <p v-if="view === 'first'">This is the first message.</p>
    <p v-else-if="view === 'second'">This is the second message.</p>
    <p v-else>This is the default message.</p>
  </div>
</template>

<script>
export default {
    data() {
        return {
            view: 'none',
        };
    },
    methods: {
        setView(view) {
            this.view = view;
        },
    },
};
</script>

<style scoped>
h1 {
    color: #42b983;
}
button {
    margin-right: 10px;
    margin-bottom: 10px;
}
</style>

Agregar la ruta en src/router/index.js


import ConditionalView from '../views/ConditionalView.vue';

const routes = [
  {
    path: '/conditional',

v-show

Muestra u oculta un elemento basado en la verdad de la expresión. A diferencia de v-if, el elemento siempre está en el DOM.

<p v-show="seen">Siempre estoy en el DOM</p>

</template>

<script>
export default {
    data() {
        return {
            view: 'none',
        };
    },
    methods: {
        setView(view) {
            this.view = view;
        },
    },
};
</script>

<style scoped>
h1 {
    color: #42b983;
}
button {
    margin-right: 10px;
    margin-bottom: 10px;
}
</style>

Métodos en Vue 3

Introducción

En este tutorial, aprenderemos sobre los métodos en Vue 3 y JavaScript. Los métodos son funciones que se pueden definir dentro de una instancia de Vue o en objetos JavaScript. Son esenciales para agregar lógica y funcionalidad a nuestras aplicaciones.

Métodos en Vue 3

En Vue 3, los métodos se definen dentro de la sección methods de una instancia del componente. Estos métodos pueden ser llamados desde el template o desde otros métodos y hooks del ciclo de vida.


        
<template>
  <div>
    <button @click="increment">Incrementar</button>
    <p>Contador: {{ counter }}</p>
  </div>
</template>

<script>
export default {
    data() {
        return {
            counter: 0
        };
    },
    methods: {
        increment() {
            this.counter++;
        }
    }
};
</script>

En este ejemplo, el método increment se define en la sección methods y se llama cuando el botón es clickeado.

Ejemplos de Métodos

Ejemplo 1: Contador en Vue 3


        
<template>
  <div>
    <button @click="increment">Incrementar</button>
    <button @click="decrement">Decrementar</button>
    <p>Contador: {{ counter }}</p>
  </div>
</template>

<script>
export default {
    data() {
        return {
            counter: 0
        };
    },
    methods: {
        increment() {
            this.counter++;
        },
        decrement() {
            this.counter--;
        }
    }
};
</script>

En este ejemplo, hemos agregado un método decrement adicional para decrementar el contador.
